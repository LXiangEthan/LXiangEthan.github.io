这个就记录了我们下载的别人的包

–save在老版本的node中是必须要加上的因为加上后可以把我们下载的包装入依赖中，现在可以省略了
下载包后会多出来一个package-lock.json文件，这个在老版本也是没有的，这个存在是为了锁定下载的包的版本，因为我们下载的包可能也依赖于别的包，这个lock是锁定了我们下载的包版本以及这个包的依赖的版本也被锁定，这样可以解决当我们或者其他作者更新后导致依赖其的包无法使用的问题

之所以我们下载一个MD5包但是出现很多包是因为md5包自己也有依赖

-dev，加上-dev会在我们的package.json文件中，把包名写在devDependencies中，而不是dependencies中，这两者的区别是前者是开发时候用的，不加dev就是上线时候用的，只是用于区分，标记位置不一样。

–save和-dev还有简写方法-S，-D

update更新，uninstall卸载

-g是全局下

npm list列举，加-g就是当前电脑下的所有npm包，不加-g就是工作区下的npm包

npm info 包名显示包的具体信息

自动按照package.json和package-lock.json下载依赖

在依赖json文件中，^和~是不一样的
^表示固定大版本好，小版本和中版本取最新的,就是^2.1.0写，最后还是2.3.0，没有^就是准确的版本2.1.0
~表示大版本号和中版本号固定，小版本取最新的
