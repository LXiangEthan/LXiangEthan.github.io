jquery只能帮助js方面的代码，而bootstrap能够实现我们写网页样式方面的缺陷，bootstrap是一个ui框架

bootstrap更加适合后台系统，因为它的样式是固定的
https://www.bootcss.com/
zepto — 9kb，当时号称9kb的zepto因为占用低，下载快，在移动端很占优势，但是当时只是把核心功能放到一个包里面，所以只有9kb，但是其他的功能还得再次下载
之所以bootstraps会把自己的包分成多个，原因就是当时的存储小，下载速度不高，所以bootstrap会把一些功能单独放一个包，当年连jquery都差点因为占用高差点活不下去了，bootstrap所有的功能就在两个包里面bootstrap.css&bootstrap.bundle.js
bootstrap的2/3版本必须依赖于jquery，但是现在5版本根本不需要jquery了
在bootstrap中有些1功能甚至都不需要js引入

==container==这个class名字必须记住，container为class的div盒子里面会自动有一个媒体查询，响应式布局
这个盒子是一个两边有留白，响应式布局，随屏幕变宽
如果要做移动端，要占满100%，就不要用container，用container-fluid，就是宽度占满100%
