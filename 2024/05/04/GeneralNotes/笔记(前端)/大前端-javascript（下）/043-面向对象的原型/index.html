在通过new出一个新对象，对象里面无论是数据还是函数，都是会在堆区里面随着新创建的对象反复创建新的，这样会比较浪费，函数如果创建多个会大量浪费空间，于是解决这个问题就有了原型的概念

在上述代码中render每次创建一个过于浪费于是为了解决问题可以使用下列原型的方案

原型写在对象外，而且函数写在原型上面可以共享内存，不用新创建内存；

那么对象的原型的函数里面，为什么对象能直接访问到，原因如下:
原型父对象=== 构造函数.prototype._proto_
创建的对象下的_proto_回去找构造函数下的原型，找到了就正常调用，找不到就报错
对象._proto_自动创建，而这个_proto_是一个隐式的

扩展–原型链概念

函数的上一代也是一个对象，这个函数是由对象创建，这个对象也有一个_proto_
在我们使用toString会先在obj1上面找，如果找不到，沿着obj1的_proto_往它的构造函数的prototype下面找，如果再找不到，继续通过_proto_往上一级对象找

使用原型使函数共用会出现覆盖的情况吗

答案是不会，如果obj1重新把getName赋值，那么，obj1就不会去找prototype下面的getName(),而是直接创建一个新的getName函数在obj1中，然后调用它，不会修改CreateObj构造函数下面prototype下的getName

但是通过
CreateObj.prototype.getName = function(){
·································
}
就能够修改

如果遇到有人偷偷修改原型下的方法为null，别犹豫，邦邦给他两拳
