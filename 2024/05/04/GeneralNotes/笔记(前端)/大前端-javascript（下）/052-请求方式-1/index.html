GET/POST
GET：明文显示传输内容，不够安全
POST：放在请求体中发送给后端，是暗文传输，相对于安全
依据后端提供的接口是哪种类型,使用哪种请求方式由后端决定
请求方式类型

get：偏向于获取数据
post：偏向于提交数据
put：偏向于更新信息（全部修改，覆盖式修改）
delete：偏向于删除信息
patch: 偏向部分修改
header: 偏向获取服务器头信息
opthion: 偏向获取服务器的设备信息
connect: 保留请求方式
这几种方式都可以实现这些功能，只是使用不同的请求类型，可以实现对某项功能的重视或者区分

给前端实现接口，最快速的方法是我们前端的nodejs后端，nodejs下面的json-server模块提供接口

安装nodejs，npm被绑定再node中
npm(node packgae manage)

npm install json-server -g
安装node的json-server模块，-g表示全局安装
json-server -v查看版本号
json-server专门提供接口的模块，基于一个json文件就可以创建很多的后端模拟接口

`json-server ./Ajax.json –watch
模拟启动后端服务器，json-server在本机的3000端口上

键盘上键可以把之前使用过的命令返回来

使用post请求json文件不响应，所以我们必须使用接口的方式获取，接口获取的数据可以使用所有请求方式
