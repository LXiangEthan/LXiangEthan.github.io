浅拷贝如果在对象中没有引用数据类型那么就可以使用，如果出现了应用数据类型则会导致浅拷贝问题，并没有创建一个新对象，只是应用了原对象的地址

直接对对象.toString( )可以得到[object,Object]
于是我们就可以使用递归来实现递归

但是这时我们发现如果我们myobj1里面的不单是对象还有数组，于是这个方法对数组不起作用，原因是数组.toString( )打印出来是[1,2,3]这样的形式，但是我们之前发现typepf 数组/对象的结果都是object，于是就可以利用这个，在js中的宗旨就是万物皆对象，可以利用这样一个方式实现数组拷贝，但是这样下来的数组并不是完全意义上的数组，知识key是索引的对象

在js中面向对象是借助原型链实现的，在这个原型链是不断继承的虽然数组没有toString()方法，但是它继承的Object对象原型是有这个方法，自然而然的就继承了下来

Object.prototype.toString()的结果是“[object Object]”，但是数字和数组的这个继承的方法则是直接转化成相应的字符串，原因是他们重写了这个原型方法

空数组也可以使用for in 循环

同时我们发现数组是有length成员，但是对象是没有的，于是我们就想到了这个方法给数组来个判断
但是这个方法好像不行，字符串也有length成员，使用这个会出现Uncaught RangeError: Maximum call stack size exceeded的报错

于是我们又有一个方法，我们直接把Object原型上的toString方法修改this来实现对array的操作，发现对于数组这个方法得到的结果是”[object,Array]“
==这样就成功实现了（太麻烦了）：==

于是还有一个比较简单的方案（JSON.stringify()/JSON.parse()）

先把对象转化成JSON字符串，再转化成JSON对象，就实现了深拷贝，但是这个方案中不能有undefined，就是这个里面如果出现了undefined，那么这个undefined属性拷贝过来就不存在了，直接会丢失这个字段

这是我们发现如果有undefined字段，我们的toString()成员方法就无法使用了，于是我们就修改这个判断条件?.
