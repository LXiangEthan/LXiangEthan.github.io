docker run创建并运行一个容器
-d 让容器在后台运行（不加-d我们使用命令可能会卡着无法继续下一步，还可以让容器一直在后台运行不影响我们的工作）
–name mysql 给容器起一个名字，必须唯一
-p 3306:3306:设置端口映射
==我们每个容器就相当于一个小型的系统有自己的文件系统，环境，还有自己的网络端口，只不过这些端口我们外部根本无法访问到，比如我们使用命令docker inspect mysql可以查看到这个容器的内部i端口，但是这个地址我们无法从外面ping到==
于是我们就可以使用端口映射，借助访问外部机器的端口，访问到容器里面的程序端口
#宿主机和mysql容器实现的端口隐射

-e KEY=VALUE：是设置环境变量
这个key和value是根据每个镜像来决定的，因为不同镜像的环境变量是不一样的

我们可以到docker hub网站里面去看这些镜像

最后一个mysql是镜像名字

不过上面这个不是完整写法，完整写法是mysql:5.7,前面是镜像名字后面是镜像版本，不写版本就默认是最新版本
