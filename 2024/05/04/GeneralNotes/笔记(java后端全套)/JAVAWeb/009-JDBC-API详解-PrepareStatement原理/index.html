预编译，会把模板编译一次，然后后面填入参数再编译参数，这样就不用每有一次sql语句就全部编译一次，模板只用编译一次，性能会高很多

这边我没有预编译的日志没有转义，我猜测因为我们的JDBC的prepareStatement如果我们不开启mysql的预编译功能，那么，prepareStatement就只是在我们写入的参数再加一个引号，来防止sql注入，一旦我们开启了mysql的预编译功能，那么我们就会借助转义的方式来防止sql注入
