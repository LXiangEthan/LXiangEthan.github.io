这一节我们就来看看ParamNameResolver类的源码来看看mybatis是如何封装这些不同的参数的

mybatis会判断方法的参数是否为多个，如果是多个，mybatis会把参数存放到一个map集合
然而mybatis会把这个map集合里面的value设置为传入的参数，key值如果我们不使用@Param注解，那么我们就默认使用arg0,arg1,param1,parm0作为键名，而到了sql语句中的占位符的时候，mybatis就会利用占位符中的数据作为map集合中的key值来取出map集合中的值，再代替sql语句中的占位符

arg0和param1指代的同一个键，arg1和param2也是
