Java中多态就是父类定义的对象被赋值子类对象，那么对象调用的方法就是子类的，但是java中成员变量不谈多态，这个对象调用的成员变量却是父类的成员变量

便于解耦合
用父类对象作为形参，可以接受一切子类对象
多态的弊端：不能使用子类的独有功能

为了解决java中多态不能使用子类的独有功能，所以我们需要类型转换
==强制类型转换可能存在的问题==：编译阶段有继承或者实现关系就可以实现强制转换，但是运行就可能出现问题，就比如兄弟类，都继承于同一个类，这个是无法强制转换的会报错：ClassCastExecption,但是编译器在编译阶段不会报错，因为它认为它们之间有继承或者实现的关系

为了防止强制转换出现错误，java官方建议我们使用instanceof来判断当前的对象的真实数据类型，再强制转换
