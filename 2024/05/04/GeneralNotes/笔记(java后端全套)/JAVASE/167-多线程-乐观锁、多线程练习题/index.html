悲观锁：害怕出现线程安全问题，在使用共享资源的时候一上来就加上锁，只要一个线程访问完毕之后再解锁，我们前面说到的都是悲观锁，线程安全，但是性能较差
乐观锁：一开始不上锁，等到出现线程安全问题的时候才开始控制上锁，线程安全，性能较好

多个线程对一个数据进行操作的时候可能发生同一个数据改变成另一个数据，但是被两个线程执行同样的操作，这也是线程安全问题，比如数10变成11。

于是就有了乐观锁

多线程练习题：

这边老师讲到了join的一个方法，我发现我对join方法的理解并不够透彻，我翻阅博客后找到了答案，那就是join是会暂停主线程的运行，方法start方法后面就实现了线程优先运行的结果，但是对于多个正在运行的其他线程，join方法并不能阻止，这样如果在使用join方法之前其他线程已经启动，join是无法实现阻塞其他线程的实现的
