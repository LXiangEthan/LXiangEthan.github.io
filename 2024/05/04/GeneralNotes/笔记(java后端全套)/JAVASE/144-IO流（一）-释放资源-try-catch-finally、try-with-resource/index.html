try-catch-finally

如果我们在使用流的时候在流关闭之前报错，那么==流占用的系统资源就无法释放==

finally代码区里面的代码无论try中是否抛出异常，最终都会执行，就算try catch代码块里面有return关键字，==我finally也一定要执行==

finally代码块里面不要使用return，否则无论怎样所有的return都是finally的返回

try-with-resource

自动调用try的括号里面的资源的close()方法
并且这个括号里面只能放资源对象，不能放普通对象
资源对象都会实现一个Closeable接口
