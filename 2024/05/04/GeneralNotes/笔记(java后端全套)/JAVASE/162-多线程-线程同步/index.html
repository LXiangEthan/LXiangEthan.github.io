同步代码块

同步代码块如何实现解决线程安全问题的呢？
不同线程进入同步代码块的时候，会根据锁对象在底层中添加一个标记，如果前面已经有线程对象有这个锁标记就不能加锁，会等待前面锁对象释放

但是对于我们使用常量字符串来代替锁对象实际上是不好用的，如果将来我们程序中出现大量线程，常量字符串在系统中永远只有一份，但是我们只希望使用同一个账户的线程加一样的锁，这样就会带来很大问题
于是在实际项目中我们最好是用this作为锁对象,如果是静态方法的锁对象，我们建议使用官方提供的：类名.class，这是一个对象一个类只有一个的对象

同步方法

同步方法执行原理：
如果是实例方法就会使用this作为锁对象，和同步代码块是一样的，如果是静态方法就是用当前 ==类名.class== 作为锁对象
这个锁是隐式的，看不到


同步代码块和同步方法相比，同步代码块锁的范围更小，==性能==就更好，但是对于现代计算机这点性能基本不值一提，同步方法的==可读性==更好
