CommonJS只是一套规范

这是一套大的规范，不止包括nodejs的模块化规范

我们可以把公共的功能 抽离成为一个单独的 js 文件 作为一个模块，默认情况下面这个模块里面的方法或者属性，外面是没法访问的。如果要让外部可以访问模块里面的方法或者属性，就必须在模块里面通过 exports 或者 module.exports 暴露属性或者方法。

暴露多个方法，如果写多个module.exports()后面赋值的会覆盖前面赋值的，只能写一个
要想导出多个方法，把方法放到一个对象里面，ES6的对象简写甚至不用区分key/value

这里导出对象相当于把我们的对象赋值给module.exports对象，然后在新文件中接收这个对象

export导出的包，必须要导出导入才能用，并不像script标签一样，在同一个html文件中引入就可以互相调用

我这里说一个之前没说的坑，如果使用mudule.exports = 方法名/对象名，不能省略mudule，但是如果exports.方法名/对象名 = 方法/对象，可以省略mudule

–save在老版本的node中是必须要加上的因为加上后可以把我们下载的包装入依赖中，现在可以省略了
下载包后会多出来一个package-lock.json文件，这个在老版本也是没有的，这个存在是为了锁定下载的包的版本，因为我们下载的包可能也依赖于别的包，这个lock是锁定了我们下载的包版本以及这个包的依赖的版本也被锁定，这样可以解决当我们或者其他作者更新后导致依赖其的包无法使用的问题

之所以我们下载一个MD5包但是出现很多包是因为md5包自己也有依赖

-dev，加上-dev会在我们的package.json文件中，把包名写在devDependencies中，而不是dependencies中，这两者的区别是前者是开发时候用的，不加dev就是上线时候用的，只是用于区分，标记位置不一样。

–save和-dev还有简写方法-S，-D

update更新，uninstall卸载

-g是全局下

npm list列举，加-g就是当前电脑下的所有npm包，不加-g就是工作区下的npm包

npm info 包名显示包的具体信息

看包是否过期，如果我们自己安装的和json文件中写的不一样就会出现报告，否则什么都没有

自动按照package.json和package-lock.json下载依赖

在依赖json文件中，^和~是不一样的
^表示固定大版本好，小版本和中版本取最新的,就是^2.1.0写，最后还是2.3.0，没有^就是准确的版本2.1.0
~表示大版本号和中版本号固定，小版本取最新的

认识Nodejs
Node.js是一个javascript运行环境。它让javascript可以开发后端程序，实现几乎其他后端语言实现的所有功能，可以与PHP、Java、Python、.NET、Ruby等后端语言平起平坐。Nodejs是基于V8引擎，V8是Google发布的开源JavaScript引擎，本身就是用于Chrome浏览器的js解释部分，但是Ryan Dahl 这哥们，鬼才般的，把这个V8搬到了服务器上，用于做服务器的软件。

NodeJs能实现超高的高并发，性能方面杠杠的

01 nodejs的特性
  • Nodejs语法完全是js语法，只要你懂js基础就可以学会Nodejs后端开发
  • NodeJs超强的高并发能力,实现高性能服务器
  • 开发周期短、开发成本低、学习成本低
02 使用 Node.js 需要了解多少 JavaScript

http://nodejs.cn/learn/how-much-javascript-do-you-need-to-know-to-use-nodejs

Node只有V8引擎，没有Blink内核，所以只能用js语法
还有一些中间层libuv可以实现一些浏览器不能实现的功能
Nodejs解析的js代码（没有浏览器安全级别的限制），提供很多系统级别的API
比如：

  • 文件的读写 (File System)

    • 进程的管理 (Process)

  • 网络通信 (HTTP/HTTPS)

    Node没有了浏览器单线程的限制，可以创建多个进程
    Nodejs和js最大区别就是Node没有了浏览器的限制
