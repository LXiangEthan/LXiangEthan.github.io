字符串模式：
?表示可选

:表示占位符
有一个例子，有时候后端需要前端往后端发送请求，比如往这个地址请求id位1的数据”http://localhost:3000/rount/1“
那么后端如何处理，难道真的给每一个商品都配置一个路由接口吗，这不科学
于是就可以使用:id占位符“/rount/:id”，就是我们只要是这样一个结构
“http://localhost:3000/rount/:id"，无论这个id值是什么，我们都会响应
但是这个占位符的数据可以变，但是结构不能修改，无论多一个/还是少一个/都不响应

+表示可以重复多次，都可以被响应

*表示可以匹配任意字符，只要其他地方满足条件就可以

回调函数在node.js中有一个亲切的叫法叫中间件
我们的回调函数需要做：验证token过期，cookie过期，查询数据库，返回内容，这样我们回调函数就会变得很臃肿

于是我们就回调函数的一个新的写法，就是我们中间件的一个栈
我们发现在get方法中的第二个回调函数可以写多个，这样想类似于promise的写法，第一个回调函数没成功执行，就不执行第二个回调函数，这就取决于第三个参数next，看回调函数的第三个参数让不让后面的回调函数执行，如果next()就可以往下执行，如果没有next()就不会往下执行

Express 是一个自身功能极简，完全是由路由和中间件构成一个的 web 开发框架：从本质上来说，一个 Express 应用就是在调用各种中间件。

中间件（Middleware） 是一个函数，它可以访问请求对象（request object (req)）, 响应对象（response object (res)）, 和 web 应用中处于请求-响应循环流程中的中间件，一般被命名为 next 的变量。

中间件的功能包括：

  • 执行任何代码。
  • 修改请求和响应对象。
  • 终结请求-响应循环。
  • 调用堆栈中的下一个中间件。

如果当前中间件没有终结请求-响应循环，则必须调用 next() 方法将控制权交给下一个中间件，否则请求就会挂起。

Express 应用可使用如下几种中间件：

  • 应用级中间件
  • 路由级中间件
  • 错误处理中间件
  • 内置中间件
  • 第三方中间件

使用可选则挂载路径，可在应用级别或路由级别装载中间件。另外，你还可以同时装在一系列中间件函数，从而在一个挂载点上创建一个子中间件栈。

（1）应用级中间件

（2）路由级中间件

路由级中间件和应用级中间件一样，只是它绑定的对象为 express.Router()。

（3）错误处理中间件

错误处理中间件和其他中间件定义类似，只是要使用 4 个参数，而不是 3 个，其签名如下： (err, req, res, next)。

（4）内置的中间件

express.static 是 Express 唯一内置的中间件。它基于 serve-static，负责在 Express 应用中提托管静态资源。每个应用可有多个静态目录。
