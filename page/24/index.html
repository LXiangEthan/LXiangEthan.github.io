Commonjs是一种模块化开发规范，但是在ES6中，我们自己就有了模块化开发规范
Nodejs起家开始就引入了Commonjs规范，这两个是绑定的，从13.2之后Node引入了ES6的模块化开发规范
但是如果像在html中script标签那种写module的方式会出现报错：SyntaxError: Cannot use import statement outside a module
这边它也告诉我们了写ES6module的方式：
Warning: To load an ES module, set "type": "module" in the package.json or use the .mjs extension.
在package.json中加”type”: “module”，或者用mjs扩展名
就算我们在json中加上”type”: “module”，仍然出现报错
Error [ERR_MODULE_NOT_FOUND]: Cannot find package 'module.js' imported from F:\code\untitled5\Test.js
原来是我们写的from后面的路径没有写完整，这个地方必须写完整路径.和/都不能少

然后我们还可以使用mjs后缀

注意：require写法的commonjs模块化规范不能和ES6混合使用

全局安装就是就算不在我们当时安装所在目录下（工作区），也可以访问到 -g
什么情况下需要全局安装，就比如nrm就需要全局安装
nrm可以修改npm下载的仓库，国外速度慢
全局安装nrm就是为了避免手动切换带来麻烦

NRM (npm registry manager)是npm的镜像源管理工具，有时候国外资源太慢，使用这个就可以快速地在 npm 源间切换。
手动切换方法： npm config set registry https://registry.npm.taobao.org

nrm -V查看版本

安装 nrm

在命令行执行命令，npm install -g nrm，全局安装nrm。

使用 nrm

执行命令 nrm ls 查看可选的源。 其中，带*的是当前使用的源，上面的输出表明当前源是官方源。

切换 nrm

如果要切换到taobao源，执行命令nrm use taobao。

测试速度

你还可以通过 nrm test 测试相应源的响应时间。

npm config get registry 查看当前使用的是哪一个仓库

cnpm，中国npm镜像
npm install -g cnpm –registry=https://registry.npmmirror.com
用中国这个下载就直接使用cnpm install 就可以了
这个是默认指定淘宝镜像下载
cnpm和npm是每10分钟同步一次，几乎就可以当成npm的替代品

还有一个yarn（jia四声）这个和npm完全无关
