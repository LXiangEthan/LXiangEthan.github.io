通过docker inspect mysql就能查看容器的挂在信息

发现mysql有一个匿名卷
这个匿名卷是用于mysql数据存储的挂载目录

mysql为什么有这个匿名目录是因为，mysql在运行过程中会不断的产生数据，如果不把它数据存储的文件夹挂在出来，那么容器内的存储占用会越来越高，这是不利的

这里我解释一下数据卷挂载，挂载就是把容器里面的数据放到外面宿主机中，数据存储在宿主机中，但是容器里面使用这个数据

docker pull 下载拉去镜像
docker images 查看本地镜像
docker rmi 删除本地镜像
docker build 构建自己的打包镜像
docker save 把本地镜像保存压缩到本地
docker load 加载压缩后的镜像
docker push 推到镜像仓库

docker run 为镜像创建对应容器并运行，如果镜像不存在会先拉取
docker stop 停止运行容器
docker start 把停止的容器启动起来

docker ps 查看当前容器的运行状态（不加参数只查看运行中的容器）
docker rm 删除本地容器
docker logs 查看日志
docker exec 进入容器内部
